# Jejakra API & Database Documentation

Backend documentation is split into two documents for different readers and use cases.

| Document | Description | Use for |
|----------|-------------|---------|
| **[API Documentation](./API-DOCUMENTATION.md)** | Endpoints, authentication, errors, environment, deployment | API testing, contract testing, learning the API |
| **[Database Design](./DATABASE-DESIGN.md)** | Schema, tables, relationships, Prisma, DB operations | Data testing, understanding the data model, DB setup and seed |

Start with the doc that matches your role: SQA doing API tests → API Documentation; data or integration tests → both; learning backend/DB → Database Design.
